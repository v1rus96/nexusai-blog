# Security Policy

## 🔒 Our Commitment to Security

At LionTechAI, we take the security of our techlion-blog project seriously. We appreciate your efforts to responsibly disclose any security vulnerabilities you may find.

## 📋 Supported Versions

We provide security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| Latest  | ✅ Yes             |
| < 1.0   | ❌ No              |

Only the latest version of techlion-blog receives security updates. Please ensure you're running the most recent release.

## 🚨 Reporting a Vulnerability

If you believe you've found a security vulnerability in our project, please follow these guidelines:

### Immediate Action Required
**DO NOT** create a public GitHub issue for security vulnerabilities.

### How to Report

1. **Email**: Send your report to **security@liontech.ai**
2. **Include the following information**:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if available)
   - Your contact information

### Our Response Process

| Timeline | Action |
|----------|--------|
| **Within 24 hours** | Acknowledge receipt of your report |
| **Within 72 hours** | Initial assessment and severity classification |
| **Within 7 days** | Detailed response with fix timeline |
| **Within 30 days** | Security patch released (for valid vulnerabilities) |

## 🏆 Recognition

We believe in recognizing security researchers who help us improve our security:

- **Hall of Fame**: Acknowledged contributors on our security page
- **Coordinated Disclosure**: We work with you on disclosure timeline
- **Credit**: Public recognition once the issue is resolved (with your permission)

## 🎯 Scope

### In Scope
- **Application Security**: XSS, CSRF, injection vulnerabilities
- **Authentication**: Session management, authorization bypass
- **Data Protection**: Information disclosure, data validation
- **Infrastructure**: Server misconfigurations affecting the application
- **Dependencies**: Known vulnerabilities in our dependencies

### Out of Scope
- **Social Engineering**: Attacks against our team members
- **Physical Security**: Attacks requiring physical access
- **DoS/DDoS**: Denial of service attacks
- **Spam**: Content/comment spam
- **Rate Limiting**: Issues that only affect availability

## 📊 Security Measures

### Automated Security
- **CodeQL Scanning**: Automated code analysis on every pull request
- **Dependency Scanning**: Weekly automated dependency vulnerability checks
- **Security Alerts**: GitHub security advisories enabled
- **Supply Chain Security**: Package integrity verification

### Development Security
- **Secure Development**: Security review process for all code changes
- **Branch Protection**: Direct commits to main branch blocked
- **Code Review**: All changes require security-aware review
- **Conventional Commits**: Structured commit messages for better tracking

### Deployment Security
- **Vercel Platform**: Leveraging Vercel's security infrastructure
- **HTTPS Everywhere**: All traffic encrypted in transit
- **Environment Isolation**: Separate staging and production environments
- **Secret Management**: Secure handling of API keys and credentials

## 🔄 Security Updates

- **Regular Updates**: Monthly security dependency updates
- **Emergency Patches**: Critical vulnerabilities patched within 24-48 hours
- **Security Newsletter**: Subscribe to our security updates
- **Changelog**: Security fixes documented in our changelog
